# AI 学习技能博客网站 PRD

## 1. 执行摘要

我们正在为 AI 学习者（开发者、产品经理）构建一个前后端分离的个人知识博客网站，用于记录和整理 AI 学习技能笔记。这个项目将解决学习内容散落各处难以检索、缺乏结构化整理的问题，使个人知识库更加有序和可复用。

---

## 2. 问题陈述

### 谁有这个问题？

AI 学习者（开发者、产品经理等）需要系统化记录学习内容。

### 问题是什么？

1. **知识散落**：学习笔记分散在 Notion、印象笔记、GitHub 等多处，没有统一管理
2. **难以检索**：当需要回顾某个 AI 技能时，难以快速找到相关笔记
3. **缺乏结构**：零散的笔记缺乏分类和关联，难以形成知识体系
4. **分享困难**：想分享给同事/社区时，需要手动整理格式

### 为什么痛苦？

- **效率损失**：每次需要相关内容时，平均花费 15-30 分钟寻找
- **知识遗忘**：学过的技能一段时间后忘记，笔记找不到了
- **重复学习**：因为找不到之前的学习记录，重复学习相同内容
- **无法复用**：无法将个人学习成果转化为可分享的内容

### 证据

- 假设：大多数个人学习者都有 3+ 个记录知识的平台
- 假设：平均每周花费 2+ 小时整理和寻找笔记

## 3. 目标用户和用户画像

### 主要用户画像：AI 学习者

- **角色**：开发者、产品经理、数据科学家
- **技术能力**：中高级（熟悉代码、有自建工具意愿）
- **目标**：系统化记录 AI 学习内容，形成可复用的知识库
- **痛点**：知识散落、难以检索、缺乏结构
- **行为**：日常在多个平台学习（文档、课程、社区），需要统一收集和整理

### 使用场景

1. 记录新学到的 AI 技能/概念
2. 回顾之前的学习笔记
3. 按类别浏览知识体系
4. 搜索特定关键词
5. 导出/分享部分内容

## 4. 战略上下文

### 业务目标

- **个人目标**：建立系统化的 AI 学习知识库，提升学习效率
- **可衡量指标**：
  - 文章数量：目标 50+ 篇
  - 分类覆盖：AI 基础、提示工程、AI 工具、实战案例等 5+ 类别
  - 检索效率：从平均 20 分钟降至 2 分钟

### 为什么是现在？

- AI 领域发展迅速，系统化整理知识的需求日益迫切
- 当前已有一定量的学习笔记，散落各处需要整合
- 现有技术栈成熟，前后端分离方案易实现和维护

### 竞争格局（可选）

- Notion/印象笔记：通用笔记工具，缺乏针对 AI 学习的结构化
- GitHub：偏向代码管理，文档展示不够友好
- 现有博客平台：缺乏知识库的系统性

## 5. 解决方案概述

### 高级描述

构建一个基于 **Astro + Markdown** 的静态博客网站：
- **内容存储**：本地 Markdown 文件，通过 Frontmatter 定义元数据
- **前端**：Astro + React（交互组件），静态生成 HTML
- **搜索**：Pagefind（静态搜索，无需后端）
- **部署**：Vercel + GitHub 自动部署

### 核心功能

1. **文章管理**
   - 创建、编辑、删除文章
   - 支持 Markdown 写作
   - 文章元数据：标题、分类、标签、创建时间、更新时间

2. **分类体系**
   - 一级分类（如：AI 基础、提示工程、AI 工具、实战案例）
   - 标签系统（如：LLM、Midjourney、Coze、Cursor）
   - 分类层级支持

3. **搜索功能**
   - 全文搜索
   - 按分类/标签筛选
   - 搜索结果高亮

4. **阅读体验**
   - 简洁美观的阅读界面
   - 代码高亮（针对技术内容）
   - 目录导航
   - 响应式设计（支持移动端）

5. **扩展功能（V2/V3）**
   - 评论系统
   - RSS 订阅
   - 暗黑模式

### 技术架构（选定方案）

```
框架：Astro 5.x + React（交互组件）
样式：Tailwind CSS
内容：本地 Markdown（*.md）+ Frontmatter
搜索：Pagefind（静态全文搜索）
部署：Vercel（GitHub 自动部署）
```

### 内容组织结构

```
content/
├── config.ts           # 分类定义
├── ai-basics/          # AI 基础
│   └── *.md
├── prompt-engineering/ # 提示工程
│   └── *.md
├── ai-tools/          # AI 工具
│   └── *.md
└── case-studies/      # 实战案例
    └── *.md
```

## 7. 用户故事和需求

### 史诗假设

我们相信，为 AI 学习者构建一个结构化知识博客网站，将显著提升知识检索效率（从 20 分钟降至 2 分钟），因为当前学习笔记散落各处缺乏统一管理。我们将通过发布后 30 天的搜索使用率和用户满意度衡量成功。

### 用户故事（Markdown 创作模式）

> 注意：由于采用本地 Markdown 文件创作，编辑流程为：本地创建 .md 文件 → 推送到 GitHub → 自动部署

**故事 1：创建文章**
作为用户，我想创建一篇新文章，所以我可以记录新的学习内容。

- [ ] 在 VS Code 中创建新的 .md 文件
- [ ] 填写 Frontmatter（title, category, tags, date）
- [ ] 使用 Markdown 语法编写正文
- [ ] 推送到 GitHub
- [ ] Vercel 自动部署，文章出现在网站

**故事 1.1：Frontmatter 模板**
```yaml
---
title: "文章标题"
description: "文章简述"
category: "ai-basics"
tags: ["llm", "入门"]
date: 2026-03-10
---
```

**故事 2：浏览文章列表**
作为用户，我想浏览所有文章，所以我可以了解知识库的概览。

- [ ] 打开网站首页，看到文章列表
- [ ] 每篇文章显示：标题、分类标签、发布日期、摘要
- [ ] 支持分页或无限滚动
- [ ] 点击文章标题进入详情页

**故事 3：按分类浏览**
作为用户，我想按分类浏览文章，所以我可以聚焦特定领域。

- [ ] 点击顶部导航的分类链接
- [ ] 页面展示该分类下的所有文章
- [ ] 显示分类名称和文章数量

**故事 4：搜索文章**
作为用户，我想搜索文章，所以我可以快速找到需要的内容。

- [ ] 在搜索框输入关键词
- [ ] 实时显示搜索结果
- [ ] 搜索结果高亮匹配关键词
- [ ] 点击结果进入文章详情

**故事 5：阅读文章**
作为用户，我想阅读文章内容，所以我可以学习知识。

- [ ] 文章详情页展示完整内容
- [ ] Markdown 正确渲染（标题、列表、代码块等）
- [ ] 代码块有语法高亮
- [ ] 侧边或顶部显示文章目录（TOC）
- [ ] 显示上/下篇文章导航

**故事 6：管理分类**
作为用户，我想管理分类，所以知识体系更有结构。

- [ ] 查看所有分类列表
- [ ] 创建新分类（名称、描述）
- [ ] 编辑分类信息
- [ ] 删除分类（确认无关联文章或转移）

### 技术需求

- 内容通过 Markdown 文件管理（Git 版本控制）
- Frontmatter 定义元数据（title, description, category, tags, date）
- Pagefind 提供静态全文搜索
- 支持代码块语法高亮（Shiki）
- 响应式布局（移动端友好）

- **Web 编辑后台**：本地 Markdown 编辑（VS Code/Obsidian）
- **用户系统/多用户**：单人使用，无需注册登录
- **评论系统**：V2 阶段功能
- **点赞/收藏**：V2 阶段功能

**MVP 包含：**

## 9. 技术依赖和风险

### 技术依赖

- **框架**：Astro 5.x（静态站点生成）
- **UI 组件**：React（搜索、暗黑模式切换等交互）
- **样式**：Tailwind CSS
- **搜索**：Pagefind
- **代码高亮**：Shiki（内置于 Astro）
- **部署**：Vercel

### 风险和缓解

| 风险 | 缓解措施 |
|------|----------|
| 静态搜索性能 | Pagefind 已优化，首次加载后可离线搜索 |
| 图片管理 | 使用外部图床，本地只存 Markdown |
| 编辑门槛 | 提供 Markdown 模板，降低上手难度 |
| 数据备份 | GitHub 仓库本身就是备份 |

### 技术选型（已确定）

- **数据存储**：Markdown 文件（版本控制友好）
- **技术栈**：Astro + React + Tailwind
- **部署**：Vercel + GitHub 自动部署

## 附录：推荐编辑工作流

### 工具推荐

1. **VS Code** + 插件：
   - `markdownlint` - Markdown 格式检查
   - `Markdown All in One` - 快捷键支持
   - `Prettier` - 代码格式化

2. **Obsidian**（可选）：
   - 双链笔记，适合知识管理
   - 可设置仓库同步

3. **Typora**（可选）：
   - 所见即所得 Markdown 编辑

### 创作流程

```
1. 本地打开项目文件夹
2. 创建 content/[category]/new-article.md
3. 复制 Frontmatter 模板，填写内容
4. 用 Markdown 写正文
5. git add → git commit → git push
6. Vercel 自动部署 → 访问网站
```
