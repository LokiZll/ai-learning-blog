## 概述

本文精选 20+ 个产品设计场景的优质提示词，帮助产品经理、设计师和开发者快速完成产品设计工作。

---

## 1. 需求分析

### 1.1 需求理解与拆解

```
请分析以下用户需求，将其拆解为具体的功能点：

需求描述：[你的需求]

请输出：
1. 核心功能（必须实现）
2. 次要功能（可以后续迭代）
3. 用户故事（3-5 个）
4. MVP 范围建议
```

### 1.2 需求优先级排序

```
我有一个需求列表，请帮我按以下维度排序：

需求列表：
1. [需求1]
2. [需求2]
3. [需求3]

请从以下维度评分（1-5分）：
- 用户价值
- 开发成本
- 实现难度
- 依赖关系

请输出排序后的需求清单及理由。
```

### 1.3 需求文档生成

```
请根据以下信息生成 PRD 文档：

产品名称：[名称]
核心功能：[描述]
目标用户：[描述]
解决痛点：[描述]

请包含：
1. 产品概述
2. 用户需求
3. 功能需求
4. 非功能需求
5. 用户流程
6. 验收标准
```

## 2. 用户画像

### 2.1 用户画像创建

```
请为 [产品名称] 创建用户画像：

已知信息：
- 目标用户：[描述]
- 使用场景：[描述]
- 核心需求：[描述]

请创建 3 个典型用户画像，包含：
- 基本信息（年龄、职业、技术水平）
- 痛点与需求
- 使用习惯
- 目标与期望
```

### 2.2 用户旅程地图

```
请为 [功能/产品] 绘制用户旅程地图：

用户：[目标用户]
核心任务：[任务描述]

请包含：
1. 各个阶段（发现→考虑→购买→使用→推荐）
2. 每个阶段的用户行为
3. 触点和情绪
4. 痛点和机会点
5. 改进建议
```

### 2.3 用户访谈问题

```
我需要设计用户访谈，请帮我生成访谈问题：

访谈目标：[目标]
用户类型：[用户描述]

请生成：
1. 开场问题（5个）
2. 核心问题（10个）
3. 深入问题（5个）
4. 结束问题（3个）
```

## 3. 功能规划

### 3.1 功能清单

```
请为 [产品/功能] 梳理完整的功能清单：

背景信息：
- 产品定位：[描述]
- 目标用户：[描述]
- 竞品：[竞品名称]

请按以下分类输出：
1. 核心功能（必须）
2. 重要功能（应该）
3. 辅助功能（可以）
4. 不做功能（暂不）

每项功能请包含：功能名称、描述、优先级
```

### 3.2 功能流程设计

```
请设计 [功能] 的用户流程：

功能描述：[描述]
用户角色：[角色]

请输出：
1. 主流程（带泳道图）
2. 分支流程
3. 异常流程
4. 边界条件
```

### 3.3 MVP 功能定义

```
请帮我定义 [产品] 的 MVP：

约束条件：
- 开发周期：[时间]
- 团队规模：[人数]
- 技术限制：[限制]

请输出：
1. MVP 必须包含的功能（不超过 5 个）
2. 每个功能的最小可实施方案
3. 迭代路线图
```

## 4. 竞品分析

### 4.1 竞品对比表

```
请分析 [你的产品] 与 [竞品A]、[竞品B] 的差异：

分析维度：
- 核心功能
- 用户体验
- 定价策略
- 市场份额
- 优劣势

请输出对比表格和关键发现。
```

### 4.2 竞品功能分析

```
请详细分析 [竞品] 的 [功能]：

请包含：
1. 功能概述
2. 用户交互流程
3. 核心亮点
4. 不足之处
5. 对我们产品的借鉴意义
```

### 4.3 市场分析

```
请分析 [市场/行业] 的现状与机会：

行业：[行业名称]
分析维度：
- 市场规模
- 增长趋势
- 主要玩家
- 用户痛点
- 机会点

请输出完整的分析报告。
```

## 5. 产品文档

### 5.1 产品介绍文档

```
请帮我写一份 [产品] 的产品介绍：

产品定位：[描述]
核心价值：[描述]
目标用户：[描述]

请包含：
1. 一句话介绍
2. 产品故事
3. 核心功能
4. 用户评价（模拟）
5. 常见问题
```

### 5.2 功能说明文档

```
请为 [功能] 编写功能说明：

功能名称：[名称]
功能]
目标描述：[描述用户：[用户]

请包含：
1. 功能概述
2. 用户场景
3. 业务流程
4. 界面原型（文字描述）
5. 规则说明
6. 异常处理
```

### 5.3 产品需求文档（PRD）模板

```
请生成一个完整的 PRD 文档模板，适用于 [产品类型]：

请包含所有标准章节，并说明每个部分需要填写的内容。
```

### 5.4 版本发布说明

```
请帮我写 [版本号] 的发布说明：

新增功能：
1. [功能1]
2. [功能2]

修复问题：
1. [问题1]
2. [问题2]

优化内容：
1. [优化1]

请按以下格式输出：
- 新增功能
- 问题修复
- 优化改进
- 已知问题
- 升级说明
```

## 6. 产品策略

### 6.1 增长策略

```
请为 [产品] 制定增长策略：

当前状态：
- 用户量：[数量]
- 增长目标：[目标]
- 可用资源：[资源]

请输出：
1. 增长飞轮
2. 关键指标
3. 策略组合（AARRR）
4. 具体行动计划
```

### 6.2 定价策略

```
请帮我设计 [产品] 的定价策略：

产品信息：
- 核心功能：[功能]
- 成本结构：[描述]
- 竞品定价：[价格]

请输出：
1. 定价模型建议
2. 价格层级设计
3. 定价策略理由
4. 促销策略建议
```

## 7. 需求评审

### 7.1 需求评审检查清单

```
请生成 [功能] 需求评审检查清单：

请从以下维度检查：
1. 完整性（功能是否覆盖所有场景）
2. 明确性（需求描述是否清晰）
3. 可行性（技术实现是否可行）
4. 优先级（是否合理）
5. 依赖关系（是否清晰）

请输出具体的检查项。
```

### 7.2 需求澄清问题

```
我需要向需求方澄清以下问题：

需求描述：[描述]
不确定的地方：
1. [问题1]
2. [问题2]

请帮我生成澄清问题清单，确保理解需求的所有细节。
```
